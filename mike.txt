What's the difference between class variable and class instance's variable



On crée un objet de type router

----

                   
router1




Ce routeur, va créer un objet de type controller : Il va l'appeler le controller1


De base, le controller1, va dès sa création piocher un objet de type vue dans une librairie : Il va l'appeler la vue1.


-----

Ensuite, il attend que le routeur(si l'utilisateur le lui demande) lui demande de piocher un objet dans une autre libraire : il va l'appeler gossip1.


Simultanément, si cela est le cas, il va demander à l'objet vue1 qu'il a crée, de lui fournir un formulaire déja prêt, à saisir.

Il va utiliser les variables venant de l'objet qu'il a crée (qui composent le formulaire) et, avec il va styliser l'autre objet qu'il a crée, l'objet gossip.



En clair, il utilise un objet qui lui permet d'avoir accès à un formulaire, et il s'en sert pour les coller dans un objet qu'il créera plus tard.







le router propose à l'utilisateur plusieurs options : -entrer, ou quitter.

Si il décide d'entrer, le routeur demande au controlleur de piocher un objet. Pour cela, le controller utilise simplement une librairie de models déja existant dans un autre fichier.



Après que le router lui ai demandé de piocher un objet, le controlleur va maintenant demander à la view de de requeter deux variables à l'auteur, et de les retourner.







--->la classe view, va générer deux gets.chomp, qui vont donner les arguments de l'objet(gossip_de_fou) créé.
